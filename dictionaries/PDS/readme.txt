September 2015 Version 1r96

This file records the changes in PDS catalog since Data Dictionary 1r66 release.

This zip file contains PDSDD.FULL and three different versions of the
data dictionary index file. The versions are for Windows,Solaris,and
Linux. You must use the appropriate index file depending on what system
you are using in order to yield correct results. 

Also,be sure that the PDSDD.FULL and the relevant index files are FTP'd
in binary mode in order for these index files to operate properly. This
is important if the .FULL file is being FTP'd from machine to machine
(Windows to Solaris,Windows to Linux,etc.)

For a Windows system,use PDSDD_WIN.IDX.

For a Solaris system,use PDSDD_SOLARIS.IDX.

For a Linux system,use PDSDD_LINUX.IDX.

These index files were generated using the program MAKE_INDEX. This 
program can be obtained from the PDS software download page at
http://pds/tools/pds-tools-package.shtml 

** NOTE **
Make sure that the PDSDD.FULL file resides in the same directory as the 
index file. Otherwise,the programs that use the index file will give a 
"cannot find data dictionary"error.

IMPORTANT
If a user wishes to create their own index file,it is necessary to
run MAKE_INDEX on the system that they intend to use the index file on.
Failure to do so will yield incorrect results.

The command line for MAKE_INDEX on Windows is:  make_index pdsdd.full.
Unix users may have to enter ./make_index pdsdd.full

The program takes about 2 seconds to execute and will create an index file
named PDSDD.IDX.  This index file contains the name of the file used as an
arguement to make_index and it may not be used with any other data dictionary
file.  If you are planning to use a data dictionary with another name run 
MAKE_INDEX for that data dictionary.
